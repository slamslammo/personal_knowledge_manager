# personal_knowledge_manager

一个面向长期学习的个人知识管理项目。目标不是简单收集资料,而是把学习过程沉淀为可复用的知识库构建范式:用 AI 协作完成资料筛选、主线梳理、可视化解释、实验验证和阶段迭代。

## 项目定位

Personal Knowledge Manager(PKM) 不是资料仓库,而是 AI 协作下的知识构建系统。每个知识领域作为一个独立板块维护,板块内部用结构化 Markdown、可视化图、可运行实验和 HTML 阅读视图沉淀学习成果。

当前第一个实验板块是 `深度学习/`。它既是一份深度学习知识库,也是验证这套 PKM 方法的样板。

## 工作原则

- **内容轨**:把知识讲透。正文由项目自己组织和编写,不堆外链,不只做资料索引。
- **工具轨**:记录展示、交互、复习、小测、学习工具等增强理解的想法。未实现的工具构想统一写入 `工具轨backlog.md`,不混入知识正文。
- **源文件与展示分离**:每个板块用 `知识库/` 维护内容源,用 `build.py` 生成可直接打开的 `知识地图.html`。

## 使用方式

打开某个知识板块的 `知识地图.html` 即可阅读。每个板块的内容源在该板块的 `知识库/` 中,修改后进入板块目录运行:

```bash
python3 build.py
```

生成的 HTML 使用相对路径引用本板块内的图片和内容,不依赖作者本机的绝对路径。

## 维护方式

项目规则见 [项目管理/01-项目维护指南.md](项目管理/01-项目维护指南.md),迭代状态见 [项目管理/02-迭代记录.md](项目管理/02-迭代记录.md)。后续新增知识领域时,应以独立板块目录与现有 `深度学习/` 并列。

阶段性修改完成后,应更新迭代记录,重新生成相关板块的 `知识地图.html`,并提交到 GitHub。

## 阶段目标

- **短期**:以 `深度学习/` 跑通知识共建流程,保证本地 HTML 可阅读、内容源可维护、实验可复现。
- **中期**:作为 GitHub 项目持续迭代,形成可公开分享的知识板块与维护规范。
- **后期**:扩展到更多知识领域,沉淀为通用的 AI 共建知识库范式。
