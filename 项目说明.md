# 项目说明 · Personal Knowledge Manager

> 本项目是一个面向长期学习的个人知识管理实验。它把"资料调研、知识主线、可视化解释、实验验证、阶段迭代"组织成可维护的开源项目。

## 1. 定位

**Personal Knowledge Manager(PKM)** 不是资料仓库,而是 AI 协作下的知识构建系统。每个知识领域作为一个独立板块维护,板块内部用结构化 Markdown、可视化图、可运行实验和 HTML 阅读视图沉淀学习成果。

当前第一个实验板块是 `深度学习/`。它既是一份深度学习知识库,也是验证这套 PKM 方法的样板。

## 2. 阶段目标

- **短期**:以 `深度学习/` 跑通知识共建流程,保证本地 HTML 可阅读、内容源可维护、实验可复现。
- **中期**:作为 GitHub 项目持续迭代,形成可公开分享的知识板块与协作规范。
- **后期**:扩展到更多知识领域,沉淀为通用的 AI 共建知识库范式。

## 3. 双轨原则

- **内容轨**:把知识讲透。正文由项目自己组织和编写,不堆外链,不只做资料索引。
- **工具轨**:记录展示、交互、复习、小测、学习工具等增强理解的想法。未实现的工具构想统一写入 `工具轨backlog.md`,不混入知识正文。

## 4. 板块结构

每个知识板块采用相同基础结构:

```text
板块名/
├── README.md              ← 板块说明
├── 知识库/                ← 内容源头:一个知识点一个 md
├── 实验/                  ← 可运行、可复现的小实验
├── assets/                ← 正文图片与 GIF
├── build.py               ← 生成 HTML 阅读视图
└── 知识地图.html          ← 浏览器阅读入口
```

板块内的 `build.py` 只依赖本板块相对路径。下载仓库后,直接打开对应板块的 `知识地图.html` 即可阅读;修改内容后,在板块目录运行 `python3 build.py` 重新生成。

## 6. 项目级文档

- `README.md`:面向 GitHub 的项目入口。
- `项目说明.md`:项目定位、结构和阶段目标。
- `项目管理/01-项目协作指南.md`:编写流程、风格、可视化、实验与维护约定。
- `项目管理/02-迭代记录.md`:阶段历史、当前状态、挂起事项和后续任务。
- `工具轨backlog.md`:跨板块的展示、交互与学习工具构想。

## 7. 维护原则

- 项目约定必须显式写在项目内,不要依赖历史聊天记录。
- 重要阶段完成后更新 `项目管理/02-迭代记录.md`。
- 新增板块时先补板块说明,再建立 `知识库/`、`实验/`、`assets/` 和生成脚本。
- 面向开源维护,避免依赖作者本机绝对路径或私有资料路径。
